## 所有喚醒情境

```
═══════════════════════════════════════════════════════════════
              深度睡眠中 → 如何喚醒？
═══════════════════════════════════════════════════════════════

深度睡眠中...
    │
    ├──────── Timer 到期（有排程時）──→ 自動喚醒
    │                                      │
    └──────── 按鈕按下（永遠可用）──→ 手動喚醒
                                           │
                                           ▼
                                    完整重開機
                                    setup() 重新執行
                                    所有 RAM 變數消失
                                    NVS + LittleFS 資料保留
                                    RTC 時鐘繼續（有漂移）
```

## 六大情境完整流程

```
═══════════════════════════════════════════════════════════════
情境 A：一般使用（DEEP_SLEEP = false）
═══════════════════════════════════════════════════════════════

  上電 → setup() → 永遠運行
       │
       ├─ STA 連上 → NTP 同步 → MQTT 發布 → Auto-logger 記錄
       │
       └─ STA 連不上 → AP 仍可用 → 手機開網頁 → /sync 同步時間
                                               → 校準 / 看數據

  ※ 不會入睡，適合有持續電源的場景


═══════════════════════════════════════════════════════════════
情境 B：深度睡眠 + STA WiFi + 有排程（全自動模式）
═══════════════════════════════════════════════════════════════

  ┌──→ Timer 喚醒（07:00）
  │        │
  │        ▼
  │    setup() → WiFi AP+STA → STA 連上 → NTP 同步（修正漂移）
  │        │
  │        ▼
  │    10 秒後 Auto-logger 記錄重量（NTP 時間戳）
  │    MQTT 每 5 秒發布
  │    Web server 可用
  │        │
  │        │ ← 10 分鐘 →
  │        ▼
  │    handleDeepSleep() 觸發
  │    getNextWakeupSeconds() → 下個排程 19:00（12 小時後）
  │    esp_sleep_enable_timer_wakeup(43200 秒)
  │    esp_deep_sleep_start()
  │        │
  │        │ ← 深度睡眠 12 小時（~10 µA）→
  │        │
  │        ▼
  │    Timer 喚醒（19:00）→ 重複上述流程
  │        │
  └────────┘

  ※ 完全無需人工介入，適合戶外電池供電


═══════════════════════════════════════════════════════════════
情境 C：深度睡眠 + 無 STA WiFi + 有排程（手機同步模式）
═══════════════════════════════════════════════════════════════

  ┌──→ Timer 或按鈕喚醒
  │        │
  │        ▼
  │    setup() → WiFi AP+STA → STA 連不上（15 秒超時）
  │        │                    STA 模式保留但未連線
  │        │                    AP 正常運作
  │        │
  │        ▼
  │    initNTP() → STA 未連線 → "[NTP] Skipped"
  │        │
  │        ▼
  │    Auto-logger 等待時間同步...
  │    每 5 秒印 "Waiting for time sync..."
  │        │
  │        ├─── 有人開手機網頁？
  │        │         │
  │        │    ┌────┴────┐
  │        │    ▼         ▼
  │        │   Yes        No
  │        │    │         │
  │        │    ▼         ▼
  │        │  /sync     10 分鐘到
  │        │  時間同步   handleDeepSleep()
  │        │    │         │
  │        │    ▼         ▼
  │        │  記錄數據  getNextWakeupSeconds()
  │        │    │       return -1（時間未同步）
  │        │    │         │
  │        │    │         ▼
  │        │    │       僅按鈕喚醒入睡
  │        │    │       （Timer 排程不會設定！）
  │        │    │
  │        │    ▼
  │        │  10 分鐘到 → 入睡
  │        │  getNextWakeupSeconds()
  │        │    │
  │        │    ├─ RTC 時間還在（/sync 設過）
  │        │    │  計算到下個排程秒數 → 設 timer
  │        │    │
  │        │    ▼
  │        │  正常入睡 + timer 喚醒
  │        │
  └────────┘

  ※ 第一次需要手機 /sync，之後 RTC 持續運行
  ※ RTC 漂移 ~±1 秒/小時，每天誤差 ~±24 秒


═══════════════════════════════════════════════════════════════
情境 D：深度睡眠 + 有排程 + 長期無 /sync（漂移累積）
═══════════════════════════════════════════════════════════════

  Day 1:  /sync 同步 → 時間準確 → 正常排程睡眠/喚醒
  Day 2:  RTC 漂移 ~±24 秒 → 排程仍然正常
  Day 7:  漂移 ~±3 分鐘 → 排程基本正常（07:03 喚醒）
  Day 30: 漂移 ~±12 分鐘 → 排程可能有感差異
  Day 90: 漂移 ~±36 分鐘 → 建議重新 /sync

  ※ 如果中途 STA 曾連上 → NTP 自動修正漂移
  ※ 如果純 AP 模式 → 建議每月開一次手機網頁 /sync


═══════════════════════════════════════════════════════════════
情境 E：深度睡眠 + 沒設排程
═══════════════════════════════════════════════════════════════

  上電 → setup() → 10 分鐘正常運行
       │
       ▼
  handleDeepSleep()
  getNextWakeupSeconds() → entryCount == 0 → return -1
       │
       ▼
  "[SLEEP] No schedule entries or time not synced — button wake-up only"
  esp_deep_sleep_start()
       │
       │ ← 無限期深度睡眠 →
       │
       ▼
  只有按鈕能喚醒！
  （沒有 timer，不會自動醒來）

  ⚠ 危險：如果沒有實體按鈕，ESP32 就「睡死了」
     只能按 RST 按鈕或重新上電才能恢復


═══════════════════════════════════════════════════════════════
情境 F：深度睡眠 + 時間從未同步（NTP 失敗 + 沒開網頁）
═══════════════════════════════════════════════════════════════

  上電 → STA 連不上 → NTP 跳過 → 沒人開網頁
       │
       ▼
  Auto-logger: 持續印 "Waiting for time sync..."
  （10 分鐘內都不會記錄任何數據）
       │
       ▼
  handleDeepSleep() → 10 分鐘到
  getNextWakeupSeconds() → getLocalTime() 失敗 → return -1
       │
       ▼
  僅按鈕喚醒入睡
  （即使有排程也無法使用，因為不知道現在幾點）

  ※ 解法：下次喚醒時用手機開網頁 → /sync → 時間同步
           之後 RTC 保持時間 → 排程正常運作
```

## 時間同步狀態轉換

```
═══════════════════════════════════════════════════════════════
              時間同步狀態機
═══════════════════════════════════════════════════════════════

  ┌─────────────┐     STA連上      ┌─────────────┐
  │             │ ──────────────→  │             │
  │  無時間      │     NTP成功      │   NTP同步    │
  │  (開機初始)  │                  │   精準 ±10ms │
  │             │ ←── STA斷線 ───  │             │
  └──────┬──────┘   下次開機STA    └──────┬──────┘
         │          連不上                 │
         │                                │
    手機開網頁                         深度睡眠
    /sync                             喚醒後
         │                                │
         ▼                                ▼
  ┌─────────────┐                  ┌─────────────┐
  │             │                  │             │
  │  /sync 同步  │    深度睡眠      │  RTC 持續    │
  │  精準 ±1s   │ ──────────────→  │  有漂移      │
  │             │    喚醒後         │  ±1s/hr     │
  │             │ ←── 手機/sync ── │             │
  └─────────────┘                  └──────┬──────┘
                                          │
                                     STA連上
                                     NTP成功
                                          │
                                          ▼
                                   ┌─────────────┐
                                   │  NTP 重新同步 │
                                   │  漂移已修正   │
                                   └─────────────┘
```

## 關鍵提醒

```
┌──────────────────────────────────────────────────────────────┐
│                        ⚠ 注意事項                            │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  1. STA 連不上不會關閉 — WiFi 保持 AP_STA 模式              │
│     AP 永遠可用，STA 只是「未連線」                          │
│                                                              │
│  2. DEEP_SLEEP_ENABLED = true + 沒排程 = 只有按鈕能喚醒     │
│     → 確保有實體按鈕，否則 ESP32 會「睡死」                  │
│                                                              │
│  3. 第一次 /sync 很重要 — 給 RTC 一個起始時間               │
│     之後即使沒手機，RTC 也能維持近似正確的時間               │
│                                                              │
│  4. RTC 漂移每天 ~±24 秒，長期建議定期 /sync 或接 STA WiFi  │
│                                                              │
│  5. 每次喚醒如果 STA 連上 → NTP 自動修正漂移（最佳情況）    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```
